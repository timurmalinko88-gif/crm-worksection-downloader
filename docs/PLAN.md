# 📋 План развития функционала CRM Downloader

---

## 🎯 **ПРИОРИТЕТ 1 - Критичное** (необходимо добавить сейчас)

### 1. Конфиг-файл (config.json)
- [ ] Вместо hardcoded переменных в коде
- [ ] Сохранять:
  - Путь `BASE_DIR`
  - Cookie `MY_COOKIE`
  - `MAX_WORKERS`
  - `MIME_MAPPER`
- **Формат:**
  ```json
  {
    "base_dir": "D:\\00_WORK_RETOUCH\\3_ZLATO.UA\\Download_Work",
    "my_cookie": "...",
    "max_workers": 5,
    "request_timeout": 30,
    "log_to_file": true
  }
  ```

### 2. Логирование в файл
- [ ] Сохранять лог каждой сессии
- [ ] Формат: `downloader_YYYY-MM-DD_HH-MM-SS.log`
- [ ] История всех загрузок
- [ ] Размер файла логов: не более 100 MB (ротация)

### 3. Повторные попытки при ошибке (Retry)
- [ ] 3 попытки на каждый файл
- [ ] Экспоненциальная задержка между попытками (1s, 2s, 4s)
- [ ] Логирование каждой попытки

### 4. Параметры командной строки (CLI)
- [ ] `python downloader.py https://url1 https://url2`
- [ ] `python downloader.py --batch urls.txt`
- [ ] `python downloader.py --config custom_config.json`
- [ ] `python downloader.py --help` - справка

### 4. Пропуск дубликатов по хешу MD5
- [ ] Сравнение с локальными файлами
- [ ] Экономия трафика
